開源工具 Cull：圖片抓取、分類與標題生成的 AI 資料集流水線

來源：Reddit r/LocalLLaMA · US/en · 閱讀原文 ↗

摘要

一位開發者開源了名為 Cull 的工具，專為 AI 圖片資料集的機器策展設計，主要解決訓練 LoRA 或建立參考圖庫時繁瑣的人工整理問題。 Cull 支援從 Civitai、X/Twitter、Reddit、Discord、Pixiv、DeviantArt、ArtStation 等超過 340 個平台抓取圖片，並整合 vision-language model 對每張圖片進行分類，支援本地 LM Studio 或雲端 Groq，輸出嚴格的 17 欄位 JSON schema，避免自由文字輸出。圖片會依品質與主題相關性評分篩選，並附上 SD prompt 的 .txt 檔與審計紀錄 .vision.json。 實際用途包含：準備 LoRA 訓練集（300 張）與 fine-tune 資料集（10 萬張），以及為無 prompt 的舊圖庫自動補上標題與分類標籤。操作介面採用 Flask + Alpine.js 儀表板，支援 ZIP 匯出直接丟入訓練器。 技術堆疊為 Python 3.10+、Playwright、gallery-dl，不需 Redis、資料庫或 Docker，MIT 授權。作者還內建了 Claude Code skill bundle 與三個子代理，讓 AI 輔助開發工具能直接理解 Cull 的核心架構。後續路線圖規劃新增更多 vision backend、CLI 模式與影片抓取功能。
