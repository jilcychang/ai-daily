Statewright：用視覺化狀態機讓 AI Agent 更可靠

來源：Hacker News — Front Page (AI filter) · US/en · 閱讀原文 ↗

摘要

前 NVIDIA、AMD 資深工程師 Ben Cochran 開發了 Statewright，一套以正式狀態機（state machine）約束 AI agent 行為的框架，旨在解決當前 agentic 系統脆弱易錯的問題。 他的核心洞察是：與其用更大的模型或更長的 prompt 來暴力解決可靠性問題，不如縮小問題的解法空間。他以 13–20B 參數的小模型測試 SWE-bench 問題，透過狀態機限制每個階段可用的工具與操作範圍——例如規劃階段只能讀取，實作階段才能編輯，測試階段只能執行測試指令。這些規則由程式碼強制執行，而非靠 prompt 提示。 實驗結果顯示，此方法在 qwen-coder、gpt-oss、gemma4 等多個模型家族上均有一致改善，甚至讓 Claude Haiku、Sonnet 表現超出預期，Opus 也能以更少 token 更穩定地完成任務。關鍵發現是：在每個步驟精準限縮上下文，遠比給模型龐大的全局上下文更有效。 Statewright 的核心是以 Rust 撰寫的狀態機引擎，並透過 MCP plugin 整合 Claude Code（未來將支援 Codex、Cursor）。使用者可透過視覺化圖形編輯器設計工作流程，清楚看到失敗路徑與重試邏輯。目前已上線並提供免費方案。
