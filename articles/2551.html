我在讓 AI 代寫之前給它的寫作規則

來源：Reddit r/singularity · US/en · 閱讀原文 ↗

摘要

一位重度 AI 寫作使用者分享了他整理出的一套提示規則，專門針對 AI 生成文字的常見缺陷。他發現即使文字技術上無誤，仍容易被識破：過度圓滑的措辭、em dash 氾濫、段落可隨意對調等。 他整理出幾條核心原則： **具體勝於精緻**：每段至少要有一個可查核的錨點，例如專有名詞、具體數字或直接引述，「各種」「有意義的改變」這類模糊詞不算。 **用普通詞**：不要追求同義詞替換，「problem」就說 problem，「we changed it」比「the implementation of the change」好。 **不要表演**：拿掉 keynote 式語氣、任務宣言句型、「Great question」「I hope this helps」等服務台口吻，從答案開始的地方開始說。 **打破規律性**：LLM 寫作最顯眼的特徵就是規律——同樣的標點節奏、三段式結構、「Not X, but Y」句型。要主動打破主導模式。 **先具體再歸納**：不要一開頭就下抽象結論，順序應是：發生了什麼、在哪出現、什麼限制、哪裡出錯、這意味著什麼。 他已將完整規則集、精簡版（約 1000 字）及迷你版（約 155 字，可直接放入 AGENTS.md 或 CLAUDE.md）公開在 GitHub 上，同時也提供了給 coding agent 用的 AGENTS.md 規範。
