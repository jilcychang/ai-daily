Build 9254 修復 TG 效能衰退並為 NVIDIA GPU 新增 PDL 支援

來源：Reddit r/LocalLLaMA · US/en · 閱讀原文 ↗

摘要

llama.cpp 最新版本 Build 9254 修復了前幾個版本出現的 Token Generation（TG）效能衰退問題，並引入了針對較新 NVIDIA GPU 的 CUDA 優化技術——Programmatic Dependent Launch（PDL）。 有用戶回報在 2x RTX 5060 Ti 16GB 的 tensor split 配置下，運行 Qwen3.6-35b-a3b-Q4_K_XL 模型可達到 3.2k PP 及 127 tg/s，比過去穩定版本還提升約 5%。 PDL 的核心概念是讓同一 CUDA stream 中的多個 kernel 能夠重疊執行，減少 kernel 啟動的延遲，效果類似 CUDA graphs，但兩者的加速效益可疊加（PDL + CG 效果最佳）。此技術僅支援運算能力 CC >= 90 的 GPU（例如 Blackwell 架構），Ada 架構不在支援範圍內，且需要在編譯時加上 `-DGGML_CUDA_PDL=ON` 旗標。 實測顯示在 RTX PRO 6000 上 TG 階段可加速約 10%，在 DGX Spark 上則有 4–5% 的提升，prefill/context 階段影響相對有限。目前已針對 gpt-oss 20b、Qwen3.5 及 Nemotron 120B Super 等模型完成 kernel 調校，幾乎所有測試模型的 TG 速度都有所提升。
