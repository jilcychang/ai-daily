用 Qwen3 27B 在 GitHub Copilot、Pi、Claude Code 和 Opencode 執行相同任務的比較

來源：Reddit r/LocalLLaMA · US/en · 閱讀原文 ↗

摘要

Reddit 用戶 sdfgeoff 進行了一項實驗，想了解 AI 程式代理（coding agent）的表現有多少來自模型本身、又有多少來自外層框架（harness）。他使用同一個模型 Qwen3 27B，分別搭配 GitHub Copilot、Pi、Claude Code 和 Opencode 四種框架，執行相同任務並比較結果。 主要發現如下：首先，Opencode 預設支援網路搜尋，在需要即時資訊的任務（如 3D 列印參數）表現明顯較好；在網頁開發任務中，Opencode 也產出了互動性強的元件。其次，GitHub Copilot 的工具呼叫機制問題最大——同樣是建立 pelican.svg 檔案，Claude Code、Pi 和 Opencode 只需 4 次 LLM 請求，Copilot 卻需要 13 次，模型一直在 edit 工具和 bash 之間反覆嘗試，導致效率低落。此外，Qwen3-vl-4 在 Opencode 上出現無限迴圈，連基本的檔案寫入都無法完成。 這項非正式測試凸顯了框架設計對模型實際表現有顯著影響，工具 schema 的品質直接決定了 LLM 能否有效呼叫工具。作者目前仍在研究如何導入自動化指標評估，以取代主觀判斷。
