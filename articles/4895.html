Microsoft 內部警告：「AI Agent 可能取代 GitHub 儲存庫角色」

來源：AI Times (Korea) · KR/ko · 閱讀原文 ↗

摘要

Microsoft 旗下的 GitHub 正面臨 AI 時代的戰略危機。儘管 AI 編程市場快速成長，GitHub 與 GitHub Copilot 的用戶數和營收均大幅提升，但 Cursor、Claude Code 等新世代 AI 編程工具正快速搶佔市場，讓 Microsoft 內部危機感日益升高。據《The Information》報導，負責統籌 GitHub 等開發者平台業務的高層 Jay Parikh 近期在內部示警：若 GitHub 無法及時調適，競爭對手不只會取代 Copilot，更可能連 GitHub 儲存庫本身的核心角色都一併取代。隨著 AI Agent 能力持續提升，程式碼的存放、協作與版本管理方式可能被重新定義，GitHub 作為開發者基礎設施的地位正受到前所未有的挑戰。這也凸顯出 Microsoft 在 AI 編程領域雖是先行者，卻面臨被更靈活的後進者彎道超車的風險。
