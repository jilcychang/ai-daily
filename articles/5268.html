在 NVIDIA DGX Spark 上跑 Qwen3.6-35B-A3B-FP8 搭配 Hermes Agent 的設定心得與改善建議

來源：Reddit r/LocalLLaMA · US/en · 閱讀原文 ↗

摘要

一位 Reddit 用戶分享了他在單台 NVIDIA DGX Spark 上部署 Qwen/Qwen3.6-35B-A3B-FP8 模型搭配 Hermes Agent 框架的完整設定，並向社群徵求意見。他使用 vLLM 的 OpenAI 相容介面，透過 Docker 容器化部署，主要設定包括：FlashInfer 作為 Attention 後端、FP8 精度的 KV Cache、最大 262,144 tokens 的超長上下文視窗，以及啟用前綴快取（Prefix Caching）與分塊預填充（Chunked Prefill）來優化長上下文效能。此外，他還啟用了 Qwen3 Next MTP 推測解碼（Speculative Decoding），每次推測 2 個 token 以提升推理速度，並保留模型的思考鏈（Thinking Chain）輸出。溫度設為 0.6、top_p 為 0.95，整體偏向較保守的生成策略。該設定目前運行穩定，但用戶希望從有類似長上下文或 Agentic 工作流部署經驗的人那裡獲得優化建議，例如記憶體使用率、並發序列數（max-num-seqs 僅設 4）等方面是否還有調整空間。這篇貼文對於想在本地端部署大型 MoE 模型並搭配 Agent 框架的開發者具有相當參考價值。
