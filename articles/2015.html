GitHub 將改為依實際 AI 使用量向 Copilot 用戶收費

來源：Ars Technica — AI · US/en · 閱讀原文 ↗

摘要

GitHub 宣布將於 6 月 1 日起，對旗下 AI 程式輔助服務 GitHub Copilot 改採依使用量計費（usage-based billing）的收費模式。現行方案讓訂閱者每月獲得固定數量的「請求次數」與「進階請求次數」配額，但 GitHub 指出，這種打包方式無法反映不同 AI 任務的實際運算成本差異——一個簡單的對話問題與一場耗時數小時的自主編程任務，目前對用戶來說費用相同。Microsoft 旗下的 GitHub 表示，公司雖然長期吸收了大量推論運算成本，但隨著 AI 運算資源需求激增，將所有「進階請求」一律視為相同成本「已不再可持續」。此次調整的目的是讓定價更貼近實際用量，同時確保 Copilot 服務在龐大需求下仍能維持財務健全。這項改變對於重度依賴 Copilot 進行複雜、長時間任務的開發者而言，可能意味著費用將顯著上升。
