OpenAI Codex 系統提示竟明確禁止「談論地精」

來源：Ars Technica — AI · US/en · 閱讀原文 ↗

摘要

OpenAI 的 Codex CLI 系統提示中，出現了一條令人費解的指令：要求最新的 GPT-5.5 模型「絕對不要談論地精（goblins）、小妖精（gremlins）、浣熊、巨魔（trolls）、食人魔（ogres）、鴿子或其他動物與生物，除非與使用者的問題明確相關」。這條禁令在長達 3,500 多字的「基礎指令」中重複出現兩次，與「不得使用 emoji 或破折號」、「不得執行 git reset --hard 等破壞性指令」等一般性規範並列。值得注意的是，同一 JSON 檔案中針對舊版模型的系統提示並不包含這條禁令，顯示這是 GPT-5.5 上線後才出現的新問題。社群媒體上也有用戶反映，GPT 近來會在毫不相關的對話中突然提起地精話題。這份提示內容是透過 OpenAI 在 GitHub 上公開的最新 Codex CLI 開源程式碼曝光，讓外界得以一窺 OpenAI 如何透過系統提示「打補丁」來修正模型的奇特行為。
