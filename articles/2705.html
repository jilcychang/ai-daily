KV Cache 量化：無知還是惡意？

來源：Reddit r/LocalLLaMA · US/en · 閱讀原文 ↗

摘要

一位在 Reddit r/LocalLLaMA 發文的工程師，使用兩張 3090 顯卡跑 Qwen-3.6 27B（FP8 safetensors）搭配 vllm，專門處理長上下文的 agentic 程式編寫任務。他發現當 KV cache 設為 FP8 量化時，模型出現大量細微錯誤、工具呼叫異常、以及推理品質明顯下降；但改回 16-bit 後，表現顯著提升。他質疑：為何社群中不斷有人將 KV cache 量化當作正經的優化方案來推薦？他認為這對低風險的聊天機器人或許還能接受，但對任何需要高可靠性的工作負載來說，KV cache 應維持全精度。他也提到 TurboQuant 同樣似乎會帶來智能損耗。此貼文引發討論，點出一個核心問題：速度與模型品質之間的取捨在實際生產環境中往往被低估，「量化 KV cache」這個流行做法可能對要求精準度的應用場景造成不可忽視的負面影響。
